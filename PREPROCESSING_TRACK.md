# Preprocessing Track

## Current Goal

Build a clean, reproducible project for a CNN + Transformer remote sensing model using true Sentinel-2 patch time series.

Main training tensor:

```text
patches: [N, T, 12, 15, 15]
```

No hand-engineered tabular collapse is used as the main dataset.

## What Was Done From The Beginning

1. Read the raw competition files and identified the main label file: `downloadedRawData/points_train_label.csv`.
2. Identified raw image folders: `downloadedRawData/region_train_1`, `downloadedRawData/region_train_2`, `downloadedRawData/region_train_3`, `downloadedRawData/region_train_4`.
3. Parsed training labels into unique point locations plus phenophase/crop labels.
4. Audited TIFF filenames across all region folders.
5. Normalized known filename formatting problems in parsing logic so malformed region separators do not break indexing.
6. Detected unsupported image products/bands and excluded them from the configured 12-band tensor.
7. Detected duplicate canonical files and kept one deterministic version per duplicate group.
8. Built a TIFF inventory by `region_id`, acquisition date, band, processing level, and path.
9. Selected one file per `region/date/band`, preferring `L2A` over `L1C`, then stable path ordering.
10. Read GeoTIFF metadata with `rasterio` because PIL reads these float TIFF values incorrectly in this environment.
11. Built region bounding boxes from raster metadata.
12. Checked each point against each region bounding box by script, not manually one by one.
13. Resolved points that matched multiple regions by choosing the best deterministic region using time steps, band count, border margin, then region id.
14. Dropped points outside all training regions.
15. Converted longitude/latitude to pixel coordinates.
16. Extracted true `15x15` patches centered on each point for each valid timestep and configured band.
17. Used edge replication when a patch crossed an image border.
18. Detected invalid pixels using the configured rule: finite and `0.0 < value <= 2.0`.
19. Preserved valid raw values and filled invalid tensor positions with `0.0` only while recording `valid_pixel_mask`.
20. Saved the clean patch dataset to `artifacts/patches_clean/train_cnn_transformer_15x15.npz`.
21. Saved train-only normalization statistics to `artifacts/normalization/train_patch_band_stats.json`.
22. Added deterministic train/validation split generation to `artifacts/splits/train_val_split.csv`.
23. Replaced weak SVG charts with PNG report charts generated by matplotlib.
24. Replaced biased sample patch selection with deterministic reservoir sampling across `random`, `invalid`, and `edge` groups.
25. Added a notebook runner: `notebooks/project_pipeline.ipynb`.
26. Added a modular CNN + Transformer baseline model scaffold.
27. Added test-time patch extraction using the same preprocessing core as training.
28. Added a generic `result.json` writer, with a clear warning that the exact submission schema still needs the official guide restored/confirmed.

## Mask Meaning

`valid_pixel_mask`:

```text
True where raw pixel value is finite and inside the configured valid range.
False where raw value was nodata/corrupt/extreme/outside valid range.
```

`band_mask`:

```text
True where that sample/timestep/band was actually read and extracted.
False where the region/date/band was missing or unreadable.
```

`time_mask`:

```text
True where the sample has a real acquisition timestep.
False where the tensor is padded to the max T length.
```

## Normalization Policy

Raw clean patch tensors are not normalized in-place.

The pipeline saves:

```text
artifacts/patches_clean/train_cnn_transformer_15x15.npz
artifacts/normalization/train_patch_band_stats.json
```

The model applies normalization at training/inference time using training-only statistics and masks invalid pixels back to zero after normalization.

## Reporting Policy

The report folder is regenerated by:

```bash
python3 scripts/preprocess_data.py --config configs/preprocessing.json
```

Important files:

```text
artifacts/preprocessing_report/preprocessing_log.md
artifacts/preprocessing_report/steps_one_by_one.md
artifacts/preprocessing_report/summary.json
artifacts/preprocessing_report/chart_manifest.json
artifacts/preprocessing_report/charts/
artifacts/preprocessing_report/samples/
```

Patch samples are deterministic with seed `42` and grouped as:

```text
random
invalid
edge
```

This avoids the old problem where the examples looked random but were actually biased toward early or invalid cells.

## Training-Time And Test-Time Alignment

Training dataset creation uses:

```text
preprocessing.dataset.build_patch_dataset(..., mode="train")
```

Final test extraction uses the same core function:

```text
preprocessing.dataset.build_patch_dataset(..., mode="test")
```

The only intended difference is that test mode can use nearest-region fallback if needed for inference robustness. The patch extraction, masks, band order, valid value thresholds, and border handling are shared.

## Known Risks

1. The exact final `result.json` schema is not confirmed because the guide folder was removed from the workspace.
2. PyTorch is installed in the local `.venv` and the baseline smoke test runs on CPU; use Colab/GPU for serious training.
3. The valid range `0.0 < value <= 2.0` is conservative for the current float reflectance-like TIFF reads, but if the platform provides scaled integer reflectance, the config must be changed.
4. Some samples contain many invalid pixels or missing date-band cells; the model must use masks and avoid treating filled zeros as normal data.
5. Center clamping can happen when a point is very near or slightly outside a date-specific raster footprint; this is logged in metadata.
